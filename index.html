Qué es el criptosistema de clave pública.

El criptosistema de clave pública, conocido como criptografía asimétrica, es un sistema criptográfico en el que se utilizan dos claves, una pública y otra privada, las cuales están matemáticamente relacionadas, pero no idénticas. La clave pública, la cual se encuentra disponible para cualquiera es utilizada para la encriptación de la información, mientras que la clave privada la cual se mantiene en secreto y solo es conocida por el propietario es utilizada para la desencriptación de la información. (GlobalSign, 2022)

El criptosistema de clave pública se basa en operaciones matemáticas que están relacionadas con problemas complejos de resolver y requieren bastante tiempo de computación para su solución. Por lo cual se hace extremadamente complejo para terceros obtener la clave privada, lo que garantiza la seguridad de la información.

Este criptosistema de clave pública es utilizado comúnmente en firmas digitales, comunicaciones seguras, autenticación, intercambio de claves, tecnología blockchain y navegación web. (Badman & Kosinski, 2024)

Cómo funciona y cuáles son sus algoritmos representativos.

Este sistema funciona mediante el uso de algoritmos criptográficos para la generación de una clave pública y una clave privada. La clave pública la cual puede ser utilizada por cualquiera, es usada para la encriptación de la información. Por otro lado, la clave privada la cual se mantiene en secreto y solo es conocida por el propietario se utiliza para la desencriptación de la información.

Estas claves se caracterizan por ser de gran longitud, lo que a su vez garantiza una mayor seguridad respecto al cifrado simétrico.

Algunos de los algoritmos más utilizados para la generación de claves públicas y privadas son:

  1. RSA (Rivest, Shamir, Adleman): Llamado así en honor a los investigadores que la propusieron, siendo el algoritmo de cifrado más extendido en el mundo, debido a su sencillez y nivel de seguridad que proporciona. El proceso consta en la generación de las dos claves a partir de dos números primos aleatorios grandes denominados p y q, con una longitud similar de bits (comprendidos entre los 1024 y 2048 bits). El proceso se encuentra basado en la factorización de estos dos números elegidos. Más información
  2. Criptografía curva elíptica (ECC): Es un método basado en las propiedades matemáticas de las curvas elípticas sobre campos finitos. Ofrece una seguridad similar al RSA con claves más pequeñas, lo que se traduce en una mayor eficiencia. Mas información
  3. Algoritmo de firma digital (DSA): Es un algoritmo diseñado específicamente para la creación de firmas digitales, garantizando la autenticidad e integridad de los mensajes.
  4. ElGamal: Es un algoritmo basado en el Intercambio de claves Diffie-Hellman, utilizado ampliamente para la transmisión segura de datos y firmas digitales. (GeeksForgeeks, 2025)

Ventajas, desventajas y diferencias con el criptosistema simétrico.

La principal diferencia que se puede encontrar entre el cifrado simétrico (criptosistema de clave privada) y el cifrado asimétrico (criptosistema de clave pública) es el uso de una sola clave en el criptosistema de clase privada, la cual, utiliza esta misma tanto para la encriptación como para la desencriptación de la información.

Si bien, el cifrado simétrico ofrece una mayor eficiencia en la encriptación de datos, el uso de solo una clave lo torna más vulnerable a que la información sea expuesta al tener que compartir la clave secreta entre las partes que se comunican, siendo posible la intercepción de esta por parte de terceros.

El criptosistema asimétrico evita esta vulnerabilidad al compartir la clave pública para la encriptación, manteniendo en todo momento la clave privada en secreto. De este modo si el atacante intercepta el mensaje encriptado, no podrá descifrar la información sin acceso a la clave privada.

Ventajas del criptosistema asimétrico

  1. No requiere el intercambio de la clave privada.
  2. Nivel de seguridad elevado al utilizar modelos matemáticos complejos
  3. Escalable y eficaz en un sistema con grandes usuarios.
  4. Facilita la confidencialidad en entornos inseguros

Desventajas del criptosistema asimétrico

  1. Menor eficiente que el cifrado simétrico.
  2. Mayor carga de recursos computacionales.
  3. No es conveniente para el cifrado de datos masivos.
  4. Ante el avance de la computación cuántica, algunos algoritmos como el RSA y el ECC podrían romperse con la implementación de algoritmos cuánticos como lo son el algoritmo de Shor dejando obsoletos los principales sistemas de cifrado. (Badman & Kosinski, 2024)

Caso práctico de aplicación (qué algoritmo, dónde y cómo usarlo).

Para el siguiente caso practico, se implementa un criptosistema de clave pública utilizando el algoritmo RSA y desarrollado en Node.js haciendo uso del modulo crypto. El proposito del codigo es cifrar un mensaje con la clave pública y descrifrarlo por medio del uso de la clave privada, asegurando asi que solo el receptor del mensaje con la clave privada puede descrifrar el contenido del mensaje

Como primer paso, se genera la clave publica y la clave privada por medio del algoritmo RSA con un tamaño de clave de 2048 bits en formato PEM 


// Importar el módulo de criptografía
const crypto = require('crypto');

// Generar claves pública y privada
const { publicKey, privateKey } = crypto.generateKeyPairSync('rsa', {
    modulusLength: 2048,
    publicKeyEncoding: {
        type: 'spki',
        format: 'pem'
    },
    privateKeyEncoding: {
        type: 'pkcs8',
        format: 'pem'
        }
    });

El segundo paso consiste en cifrar el mensaje "Este es el mensaje secreto" . Para ello el mensaje es transformado a un Buffer y cifrado con la clave pública 


// Mensaje original
const mensajeSinEncriptar = "Este es el mensaje secreto";
console.log("Mensaje sin encriptar:",  mensajeSinEncriptar);

// Encriptar con la clave pública
const mensajeEncriptado = crypto.publicEncrypt(privateKey, Buffer.from( mensajeSinEncriptar, 'utf8'));
console.log("Mensaje encriptado (base64):",  mensajeEncriptado.toString('base64'));

Finalmente, con el mensaje ya encriptado con la clave pública, se procede a la desencriptación del mensaje con la clave privada. 


// Desencriptar con la clave privada
const mensajeDesencriptado = crypto.privateDecrypt(privateKey, mensajeEncriptado);
console.log("Mensaje desencriptado:",  mensajeDesencriptado.toString('utf8'));

Observaciones y recomendaciones para implementar el criptosistema.

Si bien el criptosistema de clave pública es una de las mejores alternativas para garantizar la confidencialidad de la información, es importante tener en cuenta ciertas observaciones y recomendaciones para su implementación de forma eficiente y segura.

Observaciones

  1. Alto consumo de recursos

    2. El cifrado y descifrado con el criptosistema asimétrico requiere una mayor cantidad de recursos computacionales, lo que lo hace ineficiente para el cifrado de grandes volúmenes de datos.

  2. Limitación en el tamaño de datos a cifrar

    3. Tal y como se indicó, al ser elevado el consumo de recursos computacionales, no es recomendado cifrar archivos enteros directamente con cifrado asimétrico

Recomendaciones

  1. Utilizar el cifrado asimétrico con el cifrado simétrico.

    2. Uno de los inconvenientes que presenta el cifrado simétrico es el uso de una clave única. Sin embargo, es posible cifrar esta clave con el cifrado asimétrico, el cual permitirá aprovechar la eficiencia del cifrado simétrico garantizando la confiabilidad de la clave única.

  2. Proteger la clave privada

    3. La seguridad de la información encriptada depende de que la clave privada permanezca confidencial. Para ello es recomendado almacenarla en entornos seguros como módulos de seguridad de hardware (HSM). (FORTINET, 2022)

  3. Tamaño de las claves

    4. El uso de claves cortas perjudica la seguridad de la información encriptada, por lo cual es recomendado el uso de claves de al menos 2048 bits en RSA.

  4. Actualizaciones de las claves y algoritmos.

    5. Con el avance la computación cuántica, algunos algoritmos podrían quedar obsoletos, lo que comprometería la información encriptada con algoritmos como RSA o DSA. Por lo cual es importante mantenerse informado sobre nuevas recomendaciones y algoritmos de encriptación.

  5. Protección de ataques.

    6. Se debe tener en cuenta que, aunque el cifrado de clave público es un método de encriptación muy segura, no es inmune a amenazas cibernéticas, entre los cuales se pueden encontrar ataques de intermediario, ataques de fuerza bruta, ataques criptoanalíticos, por lo cual se debe aplicar medidas de seguridad para mitigarlos, tales como verificar la autenticidad de las claves públicas. (Dougherty, 2024)
